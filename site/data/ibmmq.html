Chapter - Installation and Versions


Upto MQ v7.0 only 1 installation per host/system/server/os image was allowed.
From version MQ v7.0.1.0, MULTIINSTANCE QUEUE MANAGER was introduced. This support HA (High Availability) feature
From version MQ v7.0.1.6 multiple installation of MQ per host/system/server/os image is allowed. It allowes upto 128 installations per host.
From version MQ v7.1, CHANNEL AUTHENTICATION RECORDS were introduced.

Chapter - Distributed Queueing

It explains how 2 systems ( can be 2 different hosts) send or receive messages through MQ protocol. The 2 systems will have 2 different queue managers.
Consider one queue manager as local or locally hosted. And another queue manager as remote or remotely hosted ( in a different system/server).
On the local queue manager we need to create a remote queue, a transmission queue and a sendig channel. And on the remote queue manager, we need to create a receiving channel and the queue where the message would be received.
The queue of the remote queue manager "local" or "local queue" to the remote queue manager.
On the local queue manager we need to create a remote queue which will have the definition of the local queue of the remote queue manager.
